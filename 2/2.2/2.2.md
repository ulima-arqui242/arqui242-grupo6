# 2.2. Requerimientos de Atributos de Calidad (Escenarios)

## Usuarios

| **Atributo de Calidad** | **Fuente del Estímulo**          | **Estímulo**                                    | **Artefacto**                 | **Entorno**                           | **Respuesta**                                                                            | **Medida de Respuesta**                                                                                    |
| ----------------------- | -------------------------------- | ----------------------------------------------- | ----------------------------- | ------------------------------------- | ---------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| Disponibilidad          | Servidor de la nube             | Alta demanda de usuarios concurrentes           | Todos los módulos             | Operación normal                      | El sistema debe redirigir a un servidor de respaldo para evitar caída de servicio         | El sistema está disponible el 99.99% del tiempo al año                                                     |
| Disponibilidad          | Servidor de la nube             | Fallo en la conexión a la base de datos         | Todos los módulos             | Operación normal                      | El sistema debe sincronizar automáticamente cuando se restablezca la conexión            | Los datos se sincronizan en menos de 1 minuto después de restablecer la conexión                           |
| Seguridad               | Usuario no autenticado          | Intento de acceso no autorizado                 | Módulo de Usuarios            | Operación normal                      | El sistema debe bloquear el acceso y registrar el intento                                | El 100% de los accesos no autorizados son bloqueados                                                       |
| Seguridad               | Sistema                         | Ataque de denegación de servicio (DDoS)         | Módulo de Usuarios            | Operación normal                      | El sistema debe mitigar los ataques DDoS utilizando firewalls                             | Los mecanismos de bloqueo deben mitigar los ataques reduciendo el tráfico malicioso en un 95% en menos de 1 minuto |
| Rendimiento             | Usuarios reportan lentitud      | Lentitud en la app                              | Módulo de Usuarios            | Operación normal                      | El sistema debe optimizar tiempos de respuesta para operaciones básicas                  | Las páginas deben cargarse concurrentemente y no superar los 5 segundos para todos                         |
| Disponibilidad          | Sistema                         | Pérdida de conectividad con un servicio externo | Módulo de Usuarios            | Operación normal                      | Se marca al servicio como no disponible hasta que se reanude su disponibilidad           | El servicio debe ser marcado como no disponible en menos de 1 segundo tras detectarse la desconexión       |
| Escalabilidad           | Crecimiento de usuarios         | Incremento en la cantidad de usuarios y datos   | Todos los módulos             | Crecimiento continuo                  | El sistema debe soportar el aumento de datos sin reducir su rendimiento                  | La aplicación debe manejar un aumento del 50% en usuarios sin afectar el tiempo de respuesta               |
| Escalabilidad           | Expansión de negocio            | Expansión a nuevas regiones                     | Todos los módulos             | Crecimiento continuo                  | El sistema debe ser escalable para soportar nuevos despliegues geográficos sin afectar el rendimiento | Puede gestionar hasta 10,000 usuarios simultáneos en regiones adicionales                                  |
| Disponibilidad          | Sistema                         | Actualización de algún servicio                 | Todos los módulos             | Operación normal                      | El servicio correspondiente entra en modo de mantenimiento sin afectar los demás         | Los demás módulos deben continuar funcionando sin interrupciones durante el mantenimiento del servicio      |
| Usabilidad              | Feedback de usuario             | Comentarios negativos sobre navegación          | Todos los módulos             | Operación normal                      | El sistema debe simplificar la interfaz en función del feedback de usuarios              | La satisfacción de usuario debe ser mayor al 90% en encuestas de usabilidad                                |
| Seguridad            | Intento de inyección SQL en la entrada de datos | Módulo de Usuarios, Módulo de Hábitos, Módulo de Amigos, Módulo de Notificaciones | Operación normal           | El sistema debe sanitizar y validar las entradas de usuario para prevenir la ejecución de consultas maliciosas. | El sistema debe detectar y bloquear el 100% de intentos de inyección SQL. | Protege la visualización e integridad de la base de datos.                                           |
