# 1.4. Listado de Stakeholders

## Modulos

### Módulo de Registro de Hábitos

| Stakeholder              | Rol                           | Responsabilidad                                                   | Interés en la Arquitectura                                                |
|--------------------------|-------------------------------|-------------------------------------------------------------------|----------------------------------------------------------------------------|
| Cliente                  | Usuario final                 | Usar la aplicación para crear, editar y eliminar hábitos.          | Asegurarse de que el módulo sea fácil de usar y útil para gestionar hábitos.|
| Analista                 | Definición de requisitos      | Definir los tipos de hábitos y configuraciones necesarias.         | Asegurar que el módulo satisfaga los requisitos del cliente.              |
| Implementador            | Desarrollo                    | Desarrollar las funcionalidades del módulo para la gestión de hábitos. | Asegurarse de que las funcionalidades estén bien implementadas.           |
| Administrador de BD       | Gestión de la base de datos   | Asegurar la integridad de los datos de hábitos en la base de datos. | Optimizar el almacenamiento de los hábitos para un acceso eficiente.      |
| Jefe de proyecto         | Gestión del proyecto          | Coordinar el desarrollo del módulo, monitoreando los plazos de entrega. | Asegurar que el desarrollo esté alineado con el cronograma.              |
| Arquitecto               | Diseño de la arquitectura     | Definir la estructura del módulo dentro del sistema global.        | Asegurar la integración del módulo con otros componentes.                |
| Especialista de despliegue | Despliegue                    | Desplegar el módulo en el entorno de producción.                   | Asegurar que el módulo esté correctamente integrado y funcione en producción. |
 

### Módulo de Seguimiento

Este módulo creara una unterfaz para marcar hábitos completados, con un resumen visual del progreso mediante un resumen visual como calendario y gráficos.

| Stakeholder              | Rol                  | Responsabilidad                                                              | Interés en la Arquitectura                                                            |
|--------------------------|----------------------|------------------------------------------------------------------------------|---------------------------------------------------------------------------------------|
| Cliente                  | Usuario final        | Proporcionar retroalimentación sobre la interfaz y el seguimiento de hábitos. | Asegurarse de que la interfaz sea clara y útil para el seguimiento de hábitos.         |
| Analista                 | Definición de requisitos | Definir las métricas y visualizaciones necesarias para el seguimiento.        | Asegurar que las métricas sean útiles para el usuario.                                |
| Implementador            | Desarrollo           | Desarrollar la interfaz para marcar hábitos completados y mostrar el progreso. | Asegurarse de que las visualizaciones sean claras y funcionales.                       |
| Responsable de pruebas   | Pruebas y calidad    | Verificar que el módulo funcione correctamente.                               | Asegurar que no haya errores en la funcionalidad de seguimiento.                      |
| Especialista de despliegue | Despliegue           | Asegurar que el módulo de seguimiento funcione correctamente en el entorno de producción. | Asegurar una implementación eficiente en los entornos de prueba y producción.         |

### Módulo de Estadísticas

Este módulo proporciona un análisis del rendimiento a través de gráficos y métricas como el porcentaje de éxito y los días consecutivos. También permite comparar diferentes hábitos para evaluar su impacto y eficacia

| Stakeholder              | Rol                  | Responsabilidad                                                            | Interés en la Arquitectura                                                             |
|--------------------------|----------------------|----------------------------------------------------------------------------|----------------------------------------------------------------------------------------|
| Cliente                  | Usuario final        | Retroalimentación sobre las métricas y gráficos.                           | Asegurar que las estadísticas sean útiles y fáciles de entender.                       |
| Analista                 | Definición de requisitos | Definir las métricas clave para analizar el rendimiento de los hábitos.     | Asegurarse de que las métricas reflejen adecuadamente el rendimiento.                  |
| Implementador            | Desarrollo           | Desarrollar la funcionalidad de gráficos y estadísticas de rendimiento.     | Asegurarse de que las visualizaciones sean eficientes y precisas.                      |
| Responsable de pruebas   | Pruebas y calidad    | Verificar que el módulo funcione correctamente.                            | Asegurar que el módulo cumpla con los estándares de calidad.                           |
| Gestor de negocio        | Dirección estratégica | Asegurar que las estadísticas ayuden a los usuarios a mejorar su rendimiento. | Evaluar cómo las estadísticas pueden mejorar la retención y satisfacción de los usuarios. |
| Evaluador                | Evaluación de calidad | Realizar pruebas de usabilidad y rendimiento de las estadísticas.           | Asegurar que las estadísticas reflejen correctamente el comportamiento de los usuarios. |

### Módulo de Notificaciones

Este módulo envía recordatorios para realizar hábitos y permite configurar notificaciones personalizables según la hora del día.

| Stakeholder              | Rol                  | Responsabilidad                                                            | Interés en la Arquitectura                                                     |
|--------------------------|----------------------|----------------------------------------------------------------------------|--------------------------------------------------------------------------------|
| Cliente                  | Usuario final        | Retroalimentación sobre la utilidad y personalización de notificaciones.   | Asegurar que las notificaciones sean precisas y útiles.                        |
| Analista                 | Definición de requisitos | Definir las notificaciones y las opciones de personalización para los usuarios. | Asegurarse de que las notificaciones se adapten a las necesidades del usuario.  |
| Implementador            | Desarrollo           | Desarrollar las funcionalidades para el envío de notificaciones.            | Asegurar que las notificaciones funcionen de manera confiable.                 |
| Administrador de BD       | Gestión de la base de datos | Asegurar la gestión eficiente de las preferencias de notificación.           | Optimizar el almacenamiento de preferencias de los usuarios.                   |
| Responsable de pruebas   | Pruebas y calidad    | Verificar que las notificaciones lleguen correctamente a los usuarios.      | Asegurar que el sistema de notificaciones esté libre de errores.               |
| Responsable de mantenimiento | Mantenimiento       | Gestionar el correcto funcionamiento de las notificaciones después del despliegue. | Asegurar que las notificaciones continúen funcionando correctamente con el tiempo. |

| Stakeholder              | Rol                  | Responsabilidad                                                             | Interés en la Arquitectura                                                    |
|--------------------------|----------------------|-----------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| Cliente                  | Usuario final        | Proporcionar retroalimentación sobre las recompensas y mensajes motivacionales. | Asegurarse de que las recompensas y mensajes motiven al usuario.              |
| Analista                 | Definición de requisitos | Definir los sistemas de recompensas y mensajes motivacionales.               | Asegurar que el módulo motive al usuario de manera efectiva.                  |
| Implementador            | Desarrollo           | Desarrollar la funcionalidad de recompensas y mensajes motivacionales.        | Asegurar que las recompensas y mensajes se entreguen correctamente.           |
| Responsable de pruebas   | Pruebas y calidad    | Verificar que las recompensas y mensajes funcionen de manera adecuada.       | Asegurar que el sistema de motivación cumpla con los estándares de calidad.   |
| Responsable de mantenimiento | Mantenimiento       | Asegurar el correcto funcionamiento del sistema de recompensas a largo plazo. | Asegurarse de que las recompensas y mensajes se actualicen sin problemas.      |
